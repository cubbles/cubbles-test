Example2: compound components with connections (partial using hook function)

cif-test-compound-outer

explanation

Description:

A compound component. (An instance from type <cif-test-compound-outer>)

Connections

The following connections are defined, outgoing from this cubbles. (This cubbles is the source component.)

  • A connection is defined with the following properties.
    • source: slot aa of this component
    • destination: slot aa of the component with member-id second
    • connection-id: outerCon1
  • A connection is defined with the following properties.
    • source: slot bb of this component
    • destination: slot bb of the component with member-id second
    • connection-id: outerCon1
  • A connection is defined with the following properties.
    • source: slot cc of this component
    • destination: slot cc of the component with member-id second
    • connection-id: outerCon3
  • A connection is defined with the following properties.
      source: slot dd of this component destination: slot dd of the component with member-id second connection-id: outerCon4

code example:


<cif-test-compound-outer
		cubx-webpackage-id="cubx.core.rte.cif-test.cif-init-test@2.2.0-SNAPSHOT" member-id="first">
	<!--
		A <cubx-core-connections> Tag for define connections between cubbles.
		The source cubble of this connections is always the parent cubble
		The <cubx-core-connections> tag can contains n  <cubx-core-connection> tags.
		 A <cubx-core-connection> Tag must  have the following mandatory attributes
		 * source - the source slot
		 * destiantion - the destination memberId and the destination slot "[destination-member-id]:[destination-slot]
		 * connection-id: the connectionId
		 and it can have the following optional attributes
		 * copy-value
		 * repeated-values
		 * hook-function
	-->
	<cubx-core-connections style="display: none;">
		<cubx-core-connection source="aa" connection-id="outerCon1"
							  destination="second:aa"></cubx-core-connection>
		<cubx-core-connection source="aa" connection-id="outerCon1"
							  destination="second:aa" type="internal"></cubx-core-connection>
		<cubx-core-connection source="bb" connection-id="outerCon2"
							  destination="second:bb"></cubx-core-connection>
		<cubx-core-connection source="cc" connection-id="outerCon3"
							  destination="second:cc"></cubx-core-connection>
		<cubx-core-connection source="dd" connection-id="outerCon4"
							  destination="second:dd"></cubx-core-connection>
	</cubx-core-connections>
</cif-test-compound-outer>

Example3: connection between elementary components which allowed repeated values

repeated-value-elementary-e

origin Content

explanation

Description:

This elementary component propagte click events (allways a true value). This component included an insertion point for origin content. This example show, how to connect two elementary components with connection attribute repeated-values=true. This is also an example for origin content

Connections

The following connections are defined, outgoing from this cubbles. (This cubbles is the source component.)

  • A connection is defined with the following properties.
    • source: slot clicked of this component
    • destination: slot clicked of the component with member-id reciver
    • connection-id: click
    • repeated-values true

Origin content

The component contains origin content, which will be placed in the component.

code example:


<repeated-value-elementary-e
		cubx-webpackage-id="cubx.core.rte.cif-test.repeated-value-test@1.2.0-SNAPSHOT" member-id="sender">
	<!--
		A <cubx-core-connections> Tag for define connections between cubbles.
		The source cubble of this connections is always the parent cubble
		The <cubx-core-connections> tag can contains n  <cubx-core-connection> tags.
		 A <cubx-core-connection> Tag must  have the following mandatory attributes
		 * source - the source slot
		 * destiantion - the destination memberId and the destination slot "[destination-member-id]:[destination-slot]
		 * connection-id: the connectionId
		 and it can have the following optional attributes
		 * copy-value
		 * repeated-values
		 * hook-function
	-->
	<cubx-core-connections>
		<cubx-core-connection source="clicked" destination="reciver:clicked" repeated-values="true"
							  connection-id="click"></cubx-core-connection>
	</cubx-core-connections>
    <!-- origin content -->
	<div style="border: solid red thin">origin Content</div>
</repeated-value-elementary-e>
