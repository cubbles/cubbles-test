compound components with connections (partial using hook function)

cif-test-compound-outer

555 0 444 "compound slot initialised" "other compound slot initialised"

explanation

Description:

Initialisation

Initialise slots of this component (<cif-test-a>)

  • slot aa will be initialised with the number value 555
  • slot bb will be initialised with the number value 444
  • slot cc will be initialised with the string value "compound slot initialised"
  • slot dd will be initialised with the string value "other compound slot initialised"

In the initialisation tag for slot aa (number value 0) is the attribute type="internal" defined. This will be deleted, because the API not allowed to initialize an internal connection.

Connections

The following connections are defined, outgoing from this cubbles. (This cubbles is the source component.)

  • A connection is defined with the following properties.
    • source: slot aa of this component
    • destination: slot aa of the component with member-id second
    • connection-id: outerCon1
  • A connection is defined with the following properties.
    • source: slot bb of this component
    • destination: slot bb of the component with member-id second
    • connection-id: outerCon1
  • A connection is defined with the following properties.
    • source: slot cc of this component
    • destination: slot cc of the component with member-id second
    • connection-id: outerCon3
  • A connection is defined with the following properties.
      source: slot dd of this component destination: slot dd of the component with member-id second connection-id: outerCon4

code example:


<cif-test-compound-outer
		cubx-webpackage-id="cubx.core.rte.cif-test.cif-init-test@2.2.0-SNAPSHOT" member-id="first">
	<!-- Example for a <cubx-core init> tag. It initialise the parent cubbles
					   The initalisation will called in order of the child <cubx-core-slot-init> tags.
					   <cubx-core init> tag can contain n <cubx-core-slot-init> tags
								   * The slot attribute define the name of the slot
								   * The slotvalue is defined in a text content. It should be a valid JSON.
				   -->
	<cubx-core-init style="display: none">
		<cubx-core-slot-init slot="aa">555</cubx-core-slot-init>
		<!-- It will be deleted, because type="internal" not allowed" -->
		<cubx-core-slot-init slot="aa" type="internal">0</cubx-core-slot-init>
		<cubx-core-slot-init slot="bb">444</cubx-core-slot-init>
		<cubx-core-slot-init slot="cc">"compound slot initialised"</cubx-core-slot-init>
		<cubx-core-slot-init slot="dd">"other compound slot initialised"</cubx-core-slot-init>
	</cubx-core-init>
	<cubx-core-connections style="display: none;">
		<cubx-core-connection source="aa" connection-id="outerCon1"
							  destination="second:aa"></cubx-core-connection>
		<cubx-core-connection source="bb" connection-id="outerCon2"
							  destination="second:bb"></cubx-core-connection>
		<cubx-core-connection source="cc" connection-id="outerCon3"
							  destination="second:cc"></cubx-core-connection>
		<cubx-core-connection source="dd" connection-id="outerCon4"
							  destination="second:dd"></cubx-core-connection>
	</cubx-core-connections>

</cif-test-compound-outer>

							  destination="third:cc"></cubx-core-connection>
		<cubx-core-connection source="dd" connection-id="outerCon8"
							  destination="third:dd"
							  hook-function="function(value,next){var newValue = value +' - added by hook function'; next(newValue);}"></cubx-core-connection>
	</cubx-core-connections>
